# Phase 1b-1 — API Routes + 真实 Agent Endpoint 接入真实引擎

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 把 Phase 0 的 toy agent endpoint 升级为对接真实 `gameRegistry` 的生产级 endpoint；补齐 matches/agents/profiles/providers 四套 CRUD Route Handlers；让一个外部 HTTP 客户端（curl / Postman / 测试脚本）能通过 API 开局、观战、查数据。**前端 UI 不在本 plan**。

**Architecture:**
- 所有 route handler 位于 `app/api/**/route.ts`，`export const runtime = 'nodejs'`
- Agent Endpoint 从 DB 查真实 agent → 调 `gameRegistry.get(gameType)` 的 botStrategy（当前 Phase，LLM 版本在 P1b-5 升级）
- Match 创建后触发首次 `tick` 走 self-fetch（真实 self-triggered tick loop 上线）
- Spectator Stream 用 Redis pub/sub 订阅 `channel:match:<id>` 转发 SSE
- 所有请求用 zod 校验

**前置条件：** P1a-1~6 全部完成（tag `phase-1a`，bot-match integration test 通过）。

**参考 spec:** 第 4 节（A2A 层）、第 7 节（GM）、第 9.1 节（页面地图里的 API 部分）。

**不做的事（留给 P1b-2~5）：**
- ❌ 任何 React / Client 组件（前端页面和组件）
- ❌ SSE spectator 页面消费端（本 plan 只做服务端广播；客户端订阅在 P1b-2+）
- ❌ LLM 调用（Agent Endpoint 内部仍走 BotStrategy；LLM 在 P1b-5 接入）

Create `app/api/agents/route.ts`:

```typescript
import { z } from 'zod'
import { listAgents, createAgent } from '@/lib/db/queries/agents'
import { findProfileById } from '@/lib/db/queries/profiles'
import { gameTypeSchema, agentKindSchema } from '@/lib/core/types'
import { log } from '@/lib/telemetry/logger'

export const runtime = 'nodejs'

const createSchema = z
  .object({
    displayName: z.string().min(1).max(80),
    gameType: gameTypeSchema,
    kind: agentKindSchema.optional(),
    profileId: z.string().min(1),
    systemPrompt: z.string().min(1).max(4000),
    avatarEmoji: z.string().max(8).nullable().optional(),
  })
  .refine(
    (data) => !(data.gameType === 'poker' && data.kind === 'moderator'),
    { message: 'poker does not support moderator kind', path: ['kind'] },
  )

export async function GET(req: Request): Promise<Response> {
  const url = new URL(req.url)
  const gameType = url.searchParams.get('gameType')
  const kind = url.searchParams.get('kind')
  const parsedGameType = gameType ? gameTypeSchema.safeParse(gameType) : null
  const parsedKind = kind ? agentKindSchema.safeParse(kind) : null
  const agents = await listAgents({
    gameType: parsedGameType?.success ? parsedGameType.data : undefined,
    kind: parsedKind?.success ? parsedKind.data : undefined,
  })
  return Response.json({ agents })
}

export async function POST(req: Request): Promise<Response> {
  let json: unknown
  try {
    json = await req.json()
  } catch {
    return Response.json({ error: 'invalid json' }, { status: 400 })
  }
  const parsed = createSchema.safeParse(json)
  if (!parsed.success) {
    return Response.json(
      { error: 'validation', details: parsed.error.flatten() },
      { status: 400 },
    )
  }
  // 验证 profile 存在
  const profile = await findProfileById(parsed.data.profileId)
  if (!profile) {
    return Response.json({ error: 'profile not found' }, { status: 404 })
  }
  const row = await createAgent(parsed.data)
  log.info('agent created', { agentId: row.id, gameType: row.gameType })
  return Response.json(row, { status: 201 })
}
```

```typescript
import { findAgentById } from '@/lib/db/queries/agents'
import { loadEnv } from '@/lib/env'

export const runtime = 'nodejs'

export async function GET(
  _req: Request,
  ctx: { params: Promise<{ agentId: string }> },
): Promise<Response> {
  const { agentId } = await ctx.params

  // 保留 P0 toy agent（便于 smoke）
  if (agentId === 'toy-poker' || agentId === 'toy-echo') {
    return legacyToyCard(agentId)
  }

  const agent = await findAgentById(agentId)
  if (!agent) {
    return Response.json({ error: `agent not found: ${agentId}` }, { status: 404 })
  }

  const env = loadEnv()
  const card = {
    protocolVersion: '0.3.0',
    name: agent.displayName,
    description: agent.systemPrompt.slice(0, 200),
    version: '1.0.0',
    url: `${env.BASE_URL}/api/agents/${agent.id}`,
    capabilities: {
      streaming: true,
      pushNotifications: false,
      stateTransitionHistory: false,
    },
    skills: [
      {
        id: `${agent.gameType}-${agent.kind}`,
        name: `${agent.gameType} ${agent.kind}`,
        description: `Plays ${agent.gameType} as ${agent.kind}`,
        tags: [agent.gameType, agent.kind],
        inputModes: ['application/json'],
        outputModes: ['application/json'],
      },
    ],
    defaultInputModes: ['application/json'],
    defaultOutputModes: ['application/json'],
    securitySchemes: {
      apiKey: {
        apiKeySecurityScheme: { location: 'header', name: 'X-Match-Token' },
      },
    },
  }
  return Response.json(card)
}

function legacyToyCard(agentId: string): Response {
  // P0 保留，用于 M2 回归
  const env = loadEnv()
  return Response.json({
    protocolVersion: '0.3.0',

    description: 'P0 toy agent',
    version: '0.0.1',
    url: `${env.BASE_URL}/api/agents/${agentId}`,
    capabilities: { streaming: true, pushNotifications: false, stateTransitionHistory: false },
    skills: [
      {
        id: 'toy', name: 'toy', description: 'toy', tags: ['toy'],
        inputModes: ['application/json'], outputModes: ['application/json'],
      },
    ],
    defaultInputModes: ['application/json'], defaultOutputModes: ['application/json'],
  })
```

```typescript
import { redis } from '@/lib/redis/client'
import { keys } from '@/lib/redis/keys'
import { isAgentParticipant } from '@/lib/db/queries/matches'

export type MatchTokenContext = {
  matchId: string
  token: string
}

/**
 * 验证 X-Match-Token 是否对应一个活动 match 且 agentId 是参与者。
 * 返回 context 或 null（401）。
 *
 * 实现策略：
 *  - Redis 里 match:<id>:token 是真值
 *  - Token → matchId 反向查找：简化方案是 token 里带 matchId 前缀，或者
 *    遍历所有 match:*:token（对 10~100 量级活动 match 可接受）
 *  - 本 Phase 用简化方案：token 头里同时带 X-Match-Id
 */
export async function validateMatchToken(
  matchId: string | null,
  token: string | null,
  agentId: string,
): Promise<MatchTokenContext | null> {
  if (!matchId || !token) return null
  const stored = await redis.get(keys.matchToken(matchId))
  if (stored !== token) return null
  const isParticipant = await isAgentParticipant(matchId, agentId)
  if (!isParticipant) return null
  return { matchId, token }
}
```

**设计说明**：增加了 `X-Match-Id` header。GM 调用 endpoint 时必须同时发 matchId + token，简化验证逻辑。spec 原本只说 `X-Match-Token`，这里做了务实调整；P2 A2A 正规化时可改。

```typescript
import { createA2AStreamResponse } from '@/lib/a2a-core/server-helpers'
import { findAgentById } from '@/lib/db/queries/agents'

import { redis } from '@/lib/redis/client'
import { keys } from '@/lib/redis/keys'
import { log } from '@/lib/telemetry/logger'

export const runtime = 'nodejs'

const messageSchema = z.object({
  message: z.object({
    messageId: z.string(),
    taskId: z.string(),
    role: z.enum(['user', 'system']),
    parts: z.array(
      z.discriminatedUnion('kind', [
        z.object({ kind: z.literal('text'), text: z.string() }),
        z.object({ kind: z.literal('data'), data: z.record(z.string(), z.unknown()) }),
      ]),
    ),
  }),
})

export async function POST(
  req: Request,
  ctx: { params: Promise<{ agentId: string }> },
): Promise<Response> {
  ensureGamesRegistered()

  const { agentId } = await ctx.params

  // 1. 保留 P0 toy 入口（integration 测试依赖）
  if (agentId === 'toy-poker' || agentId === 'toy-echo') {
    return handleLegacyToy(req, agentId)
  }

  // 2. 验证 header
  const matchId = req.headers.get('X-Match-Id')
  const token = req.headers.get('X-Match-Token')
  const tokenCtx = await validateMatchToken(matchId, token, agentId)
  if (!tokenCtx) {
    return Response.json({ error: 'unauthorized' }, { status: 401 })
  }

  // 3. 解析 body
  let json: unknown
  try {
    json = await req.json()
  } catch {
    return Response.json({ error: 'invalid json' }, { status: 400 })
  }
  const parsed = messageSchema.safeParse(json)
  if (!parsed.success) {
    return Response.json(
      { error: 'invalid body', details: parsed.error.flatten() },
      { status: 400 },
    )
  }

  // 4. 查 agent
  const agent = await findAgentById(agentId)
  if (!agent) return Response.json({ error: 'agent not found' }, { status: 404 })

  // 5. 取对应游戏模块
  const gameMod = getGame(agent.gameType as 'poker' | 'werewolf')

  // 6. 从 Redis 读当前 game state 和 validActions（由 GM 调用前保存）
  const stateRaw = await redis.get(keys.matchState(tokenCtx.matchId))
  if (!stateRaw) return Response.json({ error: 'match state missing' }, { status: 410 })
  const state = JSON.parse(stateRaw)
  const validActions = gameMod.engine.availableActions(state, agentId)

  // 7. 流式响应
  const taskId = parsed.data.message.taskId
  return createA2AStreamResponse({
    taskId,
    async execute(emit) {
      emit.statusUpdate('working')
      try {
        // Bot-only Phase：直接调 BotStrategy
        const action = gameMod.botStrategy.decide(state, validActions as unknown[])
        // 模拟一点思考文本（便于观战页看到动效）
        emit.artifactUpdate({
          parts: [{ kind: 'text', text: `[${agent.displayName}] 根据本局态势做决策...` }],
          delta: true,
        })
        await new Promise((r) => setTimeout(r, 50))

        emit.artifactUpdate({
          parts: [{ kind: 'data', data: { action, thinking: 'bot rule-based' } }],
          delta: false,
        })
        emit.statusUpdate('completed')
      } catch (err) {
        log.error('agent endpoint failed', { agentId, matchId: tokenCtx.matchId, err: String(err) })
        emit.statusUpdate('failed', String(err))
      }
    },
  })
}

async function handleLegacyToy(req: Request, agentId: string): Promise<Response> {
  const json = await req.json().catch(() => null)
  const parsed = messageSchema.safeParse(json)
  if (!parsed.success) return Response.json({ error: 'invalid body' }, { status: 400 })
  const taskId = parsed.data.message.taskId

  if (agentId === 'toy-poker') {
    return createA2AStreamResponse({

        for (const chunk of ['正在评估牌面...', ' 对手似乎很紧...', ' 决定弃牌。']) {

        emit.artifactUpdate({
          parts: [{ kind: 'data', data: { action: { type: 'fold' }, thinking: 'toy' } }],
          delta: false,
        })
        emit.statusUpdate('completed')
      },
    })
  }
  // toy-echo 略（P0 已实现；如要保留完整逻辑请把 P0 原版 copy 过来）
  return Response.json({ error: 'toy-echo temporarily disabled' }, { status: 501 })
}
```

Create `app/api/matches/[matchId]/stream/route.ts`:

```typescript
import Redis from 'ioredis'
import { loadEnv } from '@/lib/env'
import { keys } from '@/lib/redis/keys'

export const runtime = 'nodejs'
export const dynamic = 'force-dynamic'

export async function GET(
  req: Request,
  ctx: { params: Promise<{ matchId: string }> },
): Promise<Response> {
  const { matchId } = await ctx.params
  const env = loadEnv()

  const subscriber = new Redis(env.REDIS_URL)
  const channel = keys.matchChannel(matchId)
  const encoder = new TextEncoder()

  const stream = new ReadableStream<Uint8Array>({
    async start(controller) {
      // heartbeat 每 15s 防止中间代理断连
      const heartbeat = setInterval(() => {
        try {
          controller.enqueue(encoder.encode(': heartbeat\n\n'))
        } catch {
          clearInterval(heartbeat)
        }
      }, 15000)

      await subscriber.subscribe(channel)
      subscriber.on('message', (ch, msg) => {
        if (ch !== channel) return
        try {
          controller.enqueue(encoder.encode(`data: ${msg}\n\n`))
        } catch {
          /* closed */
        }
      })

      req.signal.addEventListener('abort', () => {
        clearInterval(heartbeat)
        subscriber.disconnect()
        try {
          controller.close()
        } catch { /* already closed */ }
      })
    },
  })

  return new Response(stream, {
    headers: {
      'content-type': 'text/event-stream',
      'cache-control': 'no-cache',
      connection: 'keep-alive',
      'x-accel-buffering': 'no',  // disable nginx buffering
    },
  })
}
```

## 已知漂移风险

1. **Next.js 15 `ReadableStream` + SSE 在某些版本有 buffering 问题**：如果 `curl -N` 看不到实时流，试着加 `x-accel-buffering: no` 或更新 next 到最新 patch。
2. **`drizzle-orm` delete 返回结构**：Task 3 `app/api/profiles/[profileId]/route.ts` 里 `await db.delete(...)` 返回值因驱动而异；plan 里 `void result` + 204 是通用做法，够用。
3. **ioredis subscriber 在 Next.js Route Handler 里的生命周期**：Task 8 里 `req.signal.addEventListener('abort', ...)` 应该在客户端 disconnect 时自动清理；如 Redis 连接泄露，检查这段。
4. **Self-fetch tick loop 的递归深度**：Task 7 在 handler 里 `fetch(...)` 触发下一次 tick。Node fetch 是异步 + 新连接，不会栈溢出；但极端情况下（tick 太快）可能 Node event loop 被占满。如性能差，加一个 `setTimeout(..., 10)`。
